# Examples

Directorio para proyectos ficticios y resultados sanitizados que demuestren el
uso del workflow de software design.

## Reglas

- **Nunca copies** documentos de clientes, secretos, credenciales ni
  repositorios completos.
- **Sanitiza** todos los nombres de empresas, personas, URLs y datos sensibles.
- **Mantén la estructura** real pero reemplaza el contenido con datos ficticios.

## Como crear un ejemplo

1. Copia un `software-design/` real (o crea uno ficticio).
2. Reemplaza todos los nombres propios con valores genericos (Empresa X, Usuario
   Y, etc.).
3. Elimina URLs, IPs, tokens y cualquier dato que pueda ser sensible.
4. Verifica que el ejemplo sea util para demostrar el workflow sin exponer
   informacion real.
5. Agrega un `README.md` en el directorio del ejemplo explicando que demuestra.
