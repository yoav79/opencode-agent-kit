# EPIC-002: Dashboard y Componentes UI

## Objetivo

Implementar el panel principal con estado de servicios y establecer los componentes UI reutilizables.

## Capacidades

- **CAP-004**: Dashboard con estado de servicios
- **CAP-014**: Componentes UI reutilizables
- **CAP-015**: Manejo de errores con toast
- **CAP-016**: Layout responsivo con navegación

## Requisitos

- REQ-002: Dashboard - Estado general
- REQ-003: Dashboard - Estado de servicios
- REQ-004: Dashboard - Estadísticas de correo
- REQ-005: Dashboard - Operaciones recientes

## Decisiones

- DEC-007: Manejo de errores (toasts y mensajes inline)

## Criterios de Verificación

- El dashboard muestra el estado de servicios
- La navegación lateral funciona correctamente
- Los componentes UI son reutilizables
- Los errores se muestran como toast
- La UI es responsiva

## Dependencias

- EPIC-001: Infraestructura Base y Autenticación
