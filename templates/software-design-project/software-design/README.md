# Software design

Directorio de trabajo para descubrir, especificar, disenar y validar el sistema.

## Directorios

- `docs/`: documentos vigentes y aprobados.
- `drafts/`: trabajo incompleto o pendiente de aprobacion.
- `decisions/`: decisiones arquitectonicas y sus consecuencias.
- `archive/`: versiones reemplazadas que deben conservarse.

## Fuente de verdad

`project-state.json` controla la fase, bloqueos y referencias de evidencia. No
debe utilizarse como sustituto de la documentacion narrativa.
