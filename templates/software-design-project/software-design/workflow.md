# Workflow de software design

## 1. Discovery

Salida requerida:

- problema y usuario objetivo confirmados;
- proceso actual documentado;
- criterio de exito medible;
- restricciones y exclusiones;
- preguntas y decisiones pendientes visibles.

## 2. Requirements

Salida requerida:

- requisitos funcionales identificables;
- requisitos no funcionales medibles;
- reglas de negocio y excepciones;
- criterios de aceptacion;
- trazabilidad con discovery.

## 3. Architecture

Salida requerida:

- contexto y limites;
- componentes y responsabilidades;
- datos, integraciones y contratos;
- seguridad y operacion;
- tradeoffs y decisiones registradas.

## 4. Delivery plan

Salida requerida:

- incrementos verticales demostrables;
- dependencias y secuencia;
- gates deterministas;
- estrategia de pruebas;
- riesgos y rollback.

## 5. Validation

Salida requerida:

- revision independiente;
- hallazgos cerrados o aceptados;
- cobertura requisito-componente-prueba;
- readiness explicito.
